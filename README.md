
# 🤖 Bedolaga Auto-Update Script

Автоматический скрипт для бэкапа и обновления системы Bedolaga (Telegram-бот + Кабинет + Caddy).

## ✨ Возможности

- 🔒 **Полный бэкап** базы данных, Redis, конфигов и исходного кода
- 🔄 **Автоматическое обновление** бота, кабинета и Caddy
- ✅ **Проверка здоровья** всех контейнеров после обновления
- 📊 **Цветной отчёт** с итогами работы
- 📝 **Логирование** всех действий в файл
- 🔐 **Безопасный ввод** пароля (не хранится в файле)
- ⚙️ **Интерактивная настройка** при первом запуске
- 🗑️ **Авто-ротация** старых бэкапов
- 📦 **Проверка файлов** бэкапа на целостность

## 🆕 Что нового в v2.0.0

### Telegram
- Поддержка **топиков** (`TG_THREAD_ID`) — отправка сообщений в конкретный тред группы
- **Одно сообщение** вместо двух: отчёт о бэкапе и статус версий объединены
- Проверка обновлений через **GitHub API** для бота и кабинета

### Меню настроек (пункт 4)
Вместо полного сброса — подменю из 6 пунктов:
1. Полный сброс конфига
2. Только Telegram (токен, chat_id, thread_id)
3. Только пути (бот, кабинет, caddy)
4. Резервный сервер (IP, пользователь, путь, retention)
5. Время автобэкапа (cron)
6. Кастомные файлы (открывает редактор)

### Восстановление из бэкапа (пункт 5)
- Список доступных локальных бэкапов с датами и размерами
- **Тройное подтверждение** перед восстановлением (включая ввод слова `RESTORE`)
- Полный restore: контейнеры → конфиги → PostgreSQL → Redis → кабинет → Caddy → запуск → проверка
- Уведомление в Telegram по итогу

### Защита кастомных файлов при обновлении
- Файл `/root/.bedolaga-custom-files` — список файлов, которые **не перезапишет** `git reset --hard`
- Поддерживает относительные пути (от папки проекта) и абсолютные
- Сохранение и восстановление с логированием каждого файла

### Поиск путей
- Контентные фильтры вместо проверки имени папки:
  - **Бот**: `.env` + `docker-compose.yml` + слово `bot/remnawave/telegram` внутри
  - **Кабинет**: `.env` + `package.json` + слово `cabinet/bedolaga/vite/react` внутри
  - **Caddy**: наличие `Caddyfile`
- Исключение папок с `backup` в пути
- Глубина поиска уменьшена до `maxdepth 2`
- При отказе от найденного пути — показывает **нумерованный список** всех кандидатов

## 🔍 Авто-детект путей и доменов

При первом запуске скрипт **автоматически найдёт**:

### Пути к компонентам
Сканирует: `/opt`, `/root`, `/home`, `/srv` (глубина 2)

Ищет по содержимому файлов:
- **Бот**: `.env` + `docker-compose.yml` + `bot|remnawave|telegram` внутри compose
- **Кабинет**: `.env` + `package.json` + `cabinet|bedolaga|vite|react` внутри package.json
- **Caddy**: наличие `Caddyfile`

Если найдено несколько — предложит выбрать из списка. Если не найдено — введи вручную.

### Домены для проверок
Сканирует `Caddyfile` и `.env` бота, показывает список и даёт выбрать:
- Какой домен для **кабинета**
- Какой домен для **API**

**Никаких предположений!** Скрипт спросит у тебя.


## 🚀 Быстрый старт
# 1. Скачай скрипт
```
curl -O https://raw.githubusercontent.com/Chivas600/Bedolaga-Backup-Update/main/bedolaga-update.sh
```
# 2. Сделай исполняемым
```
chmod +x bedolaga-update.sh
```
# 3. Запусти (первый раз спросит настройки)
```
sudo ./bedolaga-update.sh
```

# Запусти скрипт (из любой папки!)
```
  bedolaga-update
```

⚙️ Настройки
Где хранятся?
Файл: /root/.bedolaga-config
```
BACKUP_SERVER="IP Adress"
BACKUP_USER="root"
BACKUP_REMOTE_DIR="/root/bedolaga-backups"
BACKUP_RETENTION="7"
```

Права: 600 (только root читает)

 - Значения по умолчанию
 - Параметр
 - По умолчанию
 - Описание
 - Пользователь
 - root
 - SSH-пользователь на резервном сервере
 - Путь для бэкапов
 - /root/bedolaga-backups
 - Директория для копирования
 - Количество бэкапов
 - 7

Сколько хранить локально

Как изменить настройки?
Вариант 1: Через меню скрипта (рекомендуется)
```
bedolaga-update
# Выбери пункт 4) ⚙️ Изменить настройки сервера
```

🔐 Безопасность
- Пароль не хранится — запрашивается при каждом запуске
- Конфиг имеет права 600 (только root)
- Локальные бэкапы доступны только root
- SSH-ключи можно настроить для входа без пароля:
```
ssh-copy-id root@BACKUP_SERVER
```

🗑️ Ротация бэкапов
```
Скрипт автоматически удаляет старые бэкапы, чтобы не занимать место.
Как работает
При создании бэкапа скрипт считает количество папок
Если больше чем BACKUP_RETENTION — удаляет самые старые
Выводит сообщение с датой и размером удалённого бэкапа
```
Пример вывода
```
=== 🗑️ РОТАЦИЯ БЭКАПОВ ===

ℹ️  Храним последние 7 бэкапов...
ℹ️  Найдено 8 бэкапов. Удаляем 1 старых...
ℹ️  Удаляем бэкап от 20260401-1200 (156M)...
✅ Бэкап от 20260401-1200 удалён ✅
```
Очистка на резервном сервере
```
⚠️ Важно: Ротация работает только для локальных бэкапов.
На резервном сервере нужно чистить вручную или настроить cron.
```

🛠️ Восстановление из бэкапа
База данных
```
# 1. Останови бота
cd /opt/remnawave-bedolaga-telegram-bot
docker compose down

# 2. Восстанови том
docker run --rm \
  -v remnawave-bedolaga-telegram-bot_postgres_/target \
  -v /root/bedolaga-local-backups/bedolaga-full-backup-*/bot:/backup \
  alpine sh -c "rm -rf /target/* && tar -xzf /backup/postgres_data.tar.gz -C /target"

# 3. Запусти бота
docker compose up -d
```

Конфиги
```
cp /root/bedolaga-local-backups/bedolaga-full-backup-*/bot/.env /opt/remnawave-bedolaga-telegram-bot/
cp /root/bedolaga-local-backups/bedolaga-full-backup-*/bot/docker-compose.yml /opt/remnawave-bedolaga-telegram-bot/
```

Кабинет
```
cp /root/bedolaga-local-backups/bedolaga-full-backup-*/cabinet/.env /opt/bedolaga-cabinet/
cp -r /root/bedolaga-local-backups/bedolaga-full-backup-*/cabinet/src /opt/bedolaga-cabinet/
cd /opt/bedolaga-cabinet
npm install && npm run build
docker compose up -d --build cabinet-frontend
```

⚠️ Важные заметки

- Всегда делай бэкап перед обновлением (режим 3 или 1)
- Проверяй размер дампа БД — если 0 байт, бэкап не удался
- Храни бэкапы на отдельном сервере (скрипт копирует автоматически)
- Локальные бэкапы — последняя линия обороны
- Healthcheck кабинета использует 127.0.0.1 вместо localhost (IPv4)

🐛 Troubleshooting
Контейнер unhealthy после обновления
```
# Проверь healthcheck в docker-compose.yml кабинета
# Замени localhost на 127.0.0.1:
healthcheck:
  test: ['CMD', 'wget', '--spider', 'http://127.0.0.1:80/']
```
API endpoint не отвечает
```
Проверь Caddyfile — должен быть маршрут для hooks.yourdomain.com
Коды 404/405 — нормально (сервер работает, пути нет)
Код 000 — проблема с сетью/сервером
Ошибка при копировании бэкапа
Проверь SSH-доступ к резервному серверу
Убедись, что есть место на диске
Попробуй вручную: scp -r /root/bedolaga-local-backups/* root@IP:/path/
```

Скрипт не запускается
```
# Проверь, что symlink на месте
ls -la /usr/local/bin/bedolaga-update

# Исправь Windows-символы
sed -i 's/\r$//' /opt/bedolaga-scripts/bedolaga-update.sh

# Проверь права
chmod +x /opt/bedolaga-scripts/bedolaga-update.sh
```
