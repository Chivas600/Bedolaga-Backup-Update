# 🤖 Bedolaga Backup & Restore Script

Автоматический скрипт для бэкапа, обновления и восстановления стека **Bedolaga**:
Telegram-бот (remnawave) + Кабинет (cabinet-frontend) + Caddy + PostgreSQL + Redis.

Работает как в интерактивном режиме, так и по расписанию через cron.
При первом запуске автоматически находит все компоненты и сохраняет настройки.

---

## ✨ Возможности

- 🔒 **Полный бэкап** — конфиги, PostgreSQL (`pg_dump`), Redis, исходники кабинета, Caddyfile + копия на резервный сервер
- 🔁 **Восстановление** — из любого локального бэкапа, с тройным подтверждением
- 🔄 **Обновление** — `git pull` + пересборка контейнеров для бота и кабинета
- 🛡 **Защита кастомных файлов** — список файлов, которые не перезапишет `git reset --hard`
- 🔍 **Авто-детект** — находит пути к компонентам и реквизиты PostgreSQL по содержимому файлов
- 📡 **Telegram-уведомления** — одно сообщение с итогами бэкапа + статусом версий
- 🧵 **Поддержка топиков** — отправка в конкретный тред группы через `TG_THREAD_ID`
- 🆕 **Проверка версий** — GitHub API для бота и кабинета, сравнение с установленными
- 🗑 **Авто-ротация** — удаление старых бэкапов локально и на резервном сервере
- ⏰ **Cron-режим** — автобэкап по расписанию через флаг `--cron`
- 📊 **Отчёт** — цветной вывод + лог-файл после каждого запуска
- ⚙️ **Меню настроек** — изменить любую часть конфига без полного сброса

## 🚀 Быстрый старт

```bash
# 1. Скачай скрипт
curl -O https://raw.githubusercontent.com/Chivas600/Bedolaga-Backup-Update/main/bedolaga-update.sh

# 2. Сделай исполняемым
chmod +x bedolaga-update.sh

# 3. Создай symlink для запуска из любой папки
ln -sf /opt/bedolaga-scripts/bedolaga-update.sh /usr/local/bin/bedolaga-update

# 4. Запусти (при первом запуске пройдёт настройка)
bedolaga-update
```

```
╔════════════════════════════════════╗
║  🤖 Bedolaga: Обновление системы  ║
╚════════════════════════════════════╝

Действие:
1) 🔒 Только бэкап
2) 🔄 Только обновление
3) ⚡ Бэкап + Обновление
4) ⚙️ Настройки
5) 🔁 Восстановление из бэкапа
Выбор [1-5]:
```

| Пункт | Описание |
|-------|----------|
| **1** | Бэкап всех компонентов + копия на резервный сервер |
| **2** | `git pull` + пересборка бота и кабинета |
| **3** | Сначала бэкап, затем (с подтверждением) обновление |
| **4** | Подменю настроек — изменить только нужную часть |
| **5** | Восстановление из локального бэкапа |

## ⚙️ Настройки (пункт 4)

Вместо полного сброса — выбор что именно изменить:

```
Что изменить?
1) Всё (полный сброс)
2) Telegram (токен, chat_id, thread_id)
3) Пути (бот, кабинет, caddy)
4) Резервный сервер (IP, пользователь, путь, retention)
5) Время автобэкапа (cron)
6) Кастомные файлы (защита при обновлении)
```

| Пункт | Что делает |
|-------|-----------|
| **1** | Удаляет конфиг, при следующем запуске — полная настройка с нуля |
| **2** | Перезаписывает только Telegram-реквизиты |
| **3** | Перезапускает авто-детект путей к боту, кабинету, Caddy |
| **4** | Меняет IP/пользователя/путь/количество бэкапов на сервере |
| **5** | Устанавливает или меняет cron-расписание автобэкапа |
| **6** | Открывает `nano` для редактирования списка защищённых файлов |

### Настройка автобэкапа (cron)

```
ℹ️  Текущий cron: не настроен
⏰ Час запуска [0-23]: 3
⏰ Минута запуска [0-59]: 0
✅ Cron обновлён: 0 3 * * * /opt/bedolaga-scripts/bedolaga-update.sh --cron
```

Логи cron пишутся в `/root/bedolaga-cron.log`.

## 🔁 Восстановление

Пункт **5** главного меню запускает полное восстановление из локального бэкапа.

### Процесс

```
=== 🔁 ВОССТАНОВЛЕНИЕ ИЗ БЭКАПА ===

ℹ️  Доступные бэкапы:
  1) 20260430-1200  (156M)
  2) 20260429-0300  (154M)
  3) 20260428-0300  (153M)

📌 Выберите номер бэкапа [1-3]: 1

⚠️  Восстановление ПЕРЕЗАПИШЕТ текущие данные. Продолжить? [y/N]: y
⚠️  Папки бота и кабинета будут перезаписаны. Вы уверены? [y/N]: y
⚠️  ПОСЛЕДНИЙ ШАНС. Введите слово RESTORE для подтверждения: RESTORE
```

### Что восстанавливается

1. Остановка всех контейнеров (`docker compose down`)
2. `.env` и `docker-compose.yml` бота
3. PostgreSQL через `pg_restore --clean --if-exists`
4. Redis (tar → том)
5. `.env` и `src/` кабинета
6. `Caddyfile`
7. Запуск контейнеров (`docker compose up -d`)
8. Проверка здоровья (`do_check`)
9. Уведомление в Telegram

## 🛡 Защита кастомных файлов

При обновлении (`git reset --hard`) Git перезаписывает все файлы в папке проекта.
Чтобы сохранить кастомные файлы (логотипы, оверрайды), добавь их в список:

```
/root/.bedolaga-custom-files
```

### Формат файла

```bash
# Кастомные файлы бота (пути относительно BOT_DIR)
vpn_logo.png
assets/custom_banner.png

# Абсолютные пути тоже работают
/etc/myapp/override.conf
```

### Как работает

Перед `git reset --hard` скрипт копирует все файлы из списка во временную папку `/tmp/bedolaga-custom-TIMESTAMP/`, а после сброса — возвращает их обратно. В лог пишется каждый защищённый файл:

```
[03:00:12] 🔒 Защищён: vpn_logo.png
[03:00:14] ✅ Восстановлен: vpn_logo.png
```

Редактировать список: **Настройки → пункт 6** (открывает `nano`).

## 🔐 Безопасность

- SSH-ключ `/root/.ssh/id_backup` используется для всех `scp`/`ssh` операций
- Конфиг `/root/.bedolaga-config` имеет права `600` (только root)
- Пароли не хранятся
- Бэкапы в `/root/bedolaga-local-backups/` доступны только root

### Настройка SSH-ключа для резервного сервера

```bash
# Генерация ключа (если нет)
ssh-keygen -t ed25519 -f /root/.ssh/id_backup -N ""

# Копирование на резервный сервер
ssh-copy-id -i /root/.ssh/id_backup root@BACKUP_SERVER
```

## 🗑 Ротация бэкапов

Скрипт автоматически удаляет старые бэкапы при каждом новом бэкапе.

### Локальные бэкапы

```
ℹ️  Бэкапов: 8/7 — удаляем 1 старых...
ℹ️  Удаляем: 20260420-0300 (153M)...
✅ Удалён ✅
```

### Удалённые бэкапы

Та же логика применяется к папкам на резервном сервере — скрипт подключается по SSH и удаляет лишние.

Лимит задаётся параметром `BACKUP_RETENTION` (по умолчанию `7`). Изменить: **Настройки → пункт 4**.

## ⚠️ Важные заметки

- Всегда делай бэкап перед обновлением (пункт **3** делает это автоматически)
- `postgres.dump` создаётся через `pg_dump -Fc` — нативный формат PostgreSQL, не tar
- Авто-детект PostgreSQL читает `POSTGRES_USER` и `POSTGRES_DB` из `.env`, затем из `docker-compose.yml`
- Если дамп БД 0 байт — бэкап не удался, контейнер `remnawave_bot_db` должен быть запущен
- Healthcheck кабинета проверяет `http://127.0.0.1:80/` (не `localhost`)
- Папки с `backup` в пути исключаются из авто-детекта компонентов

## 🛠 Troubleshooting

### Контейнер `(unhealthy)` после обновления

```yaml
# В docker-compose.yml кабинета замени localhost на 127.0.0.1:
healthcheck:
  test: ['CMD', 'wget', '--spider', 'http://127.0.0.1:80/']
```

### Ошибка `pg_dump`: контейнер не найден

```bash
# Проверь имя контейнера БД
docker ps | grep postgres

# Если имя отличается от remnawave_bot_db — измени в скрипте
```

### Ошибка копирования бэкапа на сервер

```bash
# Проверь SSH-доступ
ssh -i /root/.ssh/id_backup root@BACKUP_SERVER echo ok

# Проверь место на диске
df -h
```

### Скрипт не запускается

```bash
# Проверь symlink
ls -la /usr/local/bin/bedolaga-update

# Убери Windows-символы (если редактировал на Windows)
sed -i 's/\r$//' /opt/bedolaga-scripts/bedolaga-update.sh

# Проверь права
chmod +x /opt/bedolaga-scripts/bedolaga-update.sh
```

### Авто-детект не находит компонент

Убедись что папка проекта содержит нужные файлы:
- **Бот**: `.env` + `docker-compose.yml` со словом `bot`, `remnawave` или `telegram`
- **Кабинет**: `.env` + `package.json` со словом `cabinet`, `bedolaga`, `vite` или `react`
- **Caddy**: файл `Caddyfile`

И что путь к папке не содержит слово `backup`.
